# Glossary

| Term | Meaning |
| --- | --- |
| Block | A candidate unit of ordered transactions and protocol records. |
| Finalized block | A block accepted by the protocol with enough consensus evidence to be persisted as canonical history. |
| Quorum certificate | Evidence that enough validator weight signed the required consensus step for a block. |
| PREVOTE | First voting phase used to signal support for a candidate block. |
| PRECOMMIT | Final voting phase used to form finality evidence. |
| Validator | A node identity allowed to propose or vote according to the active validator set. |
| Validator weight | Consensus voting power derived from protocol rules, currently based on active stake projection foundations. |
| Coin lot | A traceable unit of coin origin and ownership used to prove balance lineage. |
| Ledger record | Canonical record describing economic movement or protocol accounting. |
| State root | Deterministic commitment to the protocol state after execution. |
| Manifest | Versioned data-directory file recording chain identity and latest finalized state. |
| Proof of Protection | Nodo's economic and audit model for rewarding measurable protection work. |
| Treasury execution evidence | Canonical proof that a treasury action passed governance and policy checks. |
| Governance lifecycle | Proposal, voting, tally, decision, execution, and audit records for governance activity. |
| Slashing evidence | Canonical evidence of validator misbehavior that can trigger deterministic penalties. |
| Localnet | Development network profile intended for local testing. |
| Testnet candidate | Pre-public-testnet profile used to enforce stricter safety gates. |
| Mainnet | Future production network profile. It must remain blocked until readiness criteria are satisfied. |
